# Incident 001 - PowerShell Execution

## Objective

Detect suspicious PowerShell execution using Microsoft Sentinel and Sysmon telemetry.

## MITRE ATT&CK Technique

T1059.001 - Command and Scripting Interpreter: PowerShell

## Attack Simulation

Atomic Red Team test:

T1059.001-12 PowerShell Session Creation and Use

The simulation created a PowerShell session on the Windows endpoint.

## Evidence Collected

Hostname:
SOC-Windows-VM

User:
SOC-Windows-VM\azureuser

Process:
powershell.exe

Parent Process:
powershell.exe

Event Source:
Microsoft-Windows-Sysmon

Event ID:
1 - Process Creation

## Detection Logic

Sysmon process creation events were searched in Microsoft Sentinel using KQL.

## KQL Query

See detection.kql

## Investigation

The activity was traced to the azureuser account.
The command line showed PowerShell session creation activity generated by Atomic Red Team.

## Response

1. Validate whether the PowerShell activity was authorized.
2. Review command-line arguments.
3. Investigate related processes.
4. Check for persistence or credential access attempts.

## Lessons Learned

Sysmon Event ID 1 provides visibility into endpoint process execution.
PowerShell activity should be monitored because it is frequently used by attackers.
